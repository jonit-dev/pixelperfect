# Component Instructions

## Structure

- **Colocation**: Keep related styles/tests with the component if possible (or follow project structure).
- **Exports**: Named exports preferred over default exports.

## Naming

- **PascalCase**: For component filenames and function names (e.g., `Button.tsx`, `UserProfile`).
- **kebab-case**: For directories if grouping (optional, follow existing pattern).

## Props

- **Interfaces**: Define `Props` interface for every component.
- **Destructuring**: Destructure props in function signature.
- **Defaults**: Use default parameter values for optional props.
