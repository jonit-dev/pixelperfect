### Pillar 1: How-To Guides (40% of content)

Educational content that solves specific problems and naturally leads to tool usage.

**Content Types:**

- Step-by-step tutorials

Deep-dive content for specific professional audiences.

## Content Quality Standards

### Word Count Minimums

- How-To Guides: 2,500+ words
- Comparison Articles: 3,500+ words
- Quick Guides: 1,500+ words
- Industry Guides: 3,000+ words

### Required Elements for Every Post

1. **Primary keyword in H1** (naturally placed)
2. **Table of contents** (for posts 2,000+ words)
3. **Original before/after images** (minimum 3 per post)
4. **Comparison tables** (for easy scanning)
5. **FAQ section** (5-7 questions with schema markup)
6. **Clear CTA** (link to our upscaler tool)
7. **Internal links** (2-3 related posts + tool pages)
8. **External links** (2-3 authoritative sources)

### Visual Requirements

- **Before/after sliders** for image comparisons
- **Screenshots** with annotations
- **Infographics** for complex concepts
- **Video tutorials** (where possible)
- **Alt text** for all images with target keywords

## URL Structure

```
/blog/guides/[slug]       - How-to guides
/blog/comparisons/[slug]  - Comparison articles
/blog/tutorials/[slug]    - Step-by-step tutorials
/blog/industry/[slug]     - Industry-specific content
```

**Slug Guidelines:**

- Use primary keyword in URL
- Keep under 60 characters
- Use hyphens, not underscores
- Example: `/blog/guides/how-to-upscale-midjourney-images`

## Schema Markup Requirements

### All Blog Posts

```json
{
  "@type": "Article",
  "headline": "[H1]",
  "description": "[Meta description]",
  "author": { "@type": "Organization", "name": "MyImageUpscaler" },
  "datePublished": "[Date]",
  "dateModified": "[Date]"
}
```

### How-To Content

```json
{
  "@type": "HowTo",
  "name": "[Title]",
  "step": [{ "@type": "HowToStep", "name": "...", "text": "..." }]
}
```

### FAQ Sections

```json
{
  "@type": "FAQPage",
  "mainEntity": [{"@type": "Question", "name": "...", "acceptedAnswer": {...}}]
}
```

### Comparison/Review Content

```json
{
  "@type": "Review",
  "itemReviewed": {"@type": "SoftwareApplication", "name": "..."},
  "author": {...},
  "reviewRating": {...}
}
```

## Writing Guidelines

### Tone & Voice

- **Helpful** - Focus on solving problems
- **Expert** - Demonstrate deep knowledge
- **Accessible** - Explain technical concepts simply
- **Action-oriented** - Clear steps, actionable advice
- **Honest** - Acknowledge limitations, don't oversell

### SEO Writing Rules

1. Primary keyword in first 100 words
2. Primary keyword in H1, one H2, and conclusion
3. Secondary keywords in other H2s
4. Natural keyword density (1-2%)
5. LSI keywords throughout body
6. Answer featured snippet questions directly

### Avoid

- Keyword stuffing
- Thin content (<1,000 words for guides)
- Duplicate content
- Generic stock photos
- Claims without evidence
- Outdated information
